What is Version Control System?

Version Control System is a kind of database. It lets you save a project snapshot at any time.

Every change made to the project files is tracked, along with-

  • Who made the change.
  • Why the change was made.

Later, the Version Control System shows exactly how an older version differs from the previous one.

Why version control system is needed?

1. Collaboration

Without a version control system, developers working on the same project from different regions will struggle to collaborate.

2. Storing Versions

Managing multiple project versions on your disk can be challenging due to the large amount of data to store and organize.

3. Restoring Previous Versions

Version control systems let developers revert to a previous working version if mistakes occur.

4. Backup

If the system or disk fails without a backup, all files will be lost, so a version control system is essential.

Types of version control system

  • Local version control system
  • Central version control system
  • Distributed version control system

What is Git?

  • Git is a popular distributed version control system for software development.
  • It enables developers to collaborate on a project.
  • Can efficiently track and manage code changes.

How Git works?

Git stores data as snapshots of a mini filesystem. With each commit, it captures the current state of your files and saves a reference.

A Git project has three sections-

  • Working Directory (a single version checkout).
  • Staging Area (index for the next commit).
  • Git Repository (stores metadata and object database).

What is GitHub?

GitHub is a web platform for hosting projects using Git version control.

Git vs GitHub

Git GitHub
Git is a distributed version control system. GitHub is a platform for hosting Git repositories.
It tracks and manages file changes. It hosts and shares repositories.
Developers use Git locally on their computers. Developers use GitHub to store and share their Git repositories remotely.
It works offline on your computer. It syncs and collaborates remotely.
It is command-line-based. It offers a web interface.
Git is open source. GitHub provides free and paid hosting plans.
Git commands: git init, git add. git commit, git branch, etc. GitHub features: pull requests, issue tracking, repository forks, and code discussions.
